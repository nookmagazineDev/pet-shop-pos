  PetShop POS — Print Server คู่มือการใช้งาน
================================================================

Print Server คืออะไร?
-----------------------
โปรแกรมเล็กๆ ที่รันบนคอมพิวเตอร์ในร้าน
รับคำสั่งพิมพ์จากเว็บแอป แล้วส่งตรงไปยังเครื่องพิมพ์ thermal
ผ่านเครือข่าย LAN / WiFi โดยไม่ต้องติดตั้ง driver เลย

ใช้ได้จาก: คอมพิวเตอร์, แท็บเล็ต, มือถือ — ขอแค่ WiFi เดียวกัน


================================================================
  ความต้องการของระบบ
================================================================

- Node.js เวอร์ชัน 14 ขึ้นไป
  ดาวน์โหลดได้ที่: https://nodejs.org  (เลือก LTS)

- เครื่องพิมพ์ Thermal ที่รองรับ ESC/POS ผ่าน TCP Port 9100
  เช่น Epson TM-T82, Gprinter, Xprinter, Bixolon, Star ฯลฯ

- เครื่องพิมพ์และคอมที่รัน Print Server ต้องอยู่ใน WiFi / LAN เดียวกัน


================================================================
  ติดตั้งครั้งแรก (ทำแค่ครั้งเดียว)
================================================================

1. เปิด Command Prompt หรือ PowerShell
2. เข้าไปที่โฟลเดอร์ print-server
   พิมพ์:  cd "C:\...\pet-shop-pos\print-server"
3. ติดตั้ง package:
   พิมพ์:  npm install
4. รอจนเสร็จ (ครั้งแรกอาจใช้เวลา 1-2 นาที)


================================================================
  วิธีเปิดใช้งานทุกวัน
================================================================

ดับเบิลคลิกที่ไฟล์:
  เริ่มระบบปริ้นเตอร์.bat  (อยู่ในโฟลเดอร์หลักของโปรเจกต์)

หน้าต่างดำจะขึ้นข้อความแบบนี้:

  ✅  Print Server เริ่มทำงานแล้ว!
      Local  : http://localhost:3001
      Network: http://192.168.1.5:3001  ← ใช้ IP นี้บนมือถือ

** เก็บ IP บรรทัด Network ไว้ใช้ตั้งค่าในแอป **
** ห้ามปิดหน้าต่างนี้ระหว่างใช้งาน **


================================================================
  ตั้งค่าในเว็บแอป (ทำครั้งแรกหรือเมื่อเปลี่ยนเครือข่าย)
================================================================

1. เปิดเว็บแอป PetShop POS
2. ไปที่เมนู "ตั้งค่าปริ้นเตอร์"
3. กรอกข้อมูล:

   IP เครื่องพิมพ์
     → ใส่ IP ของ Thermal Printer เช่น 192.168.1.100
     → หาได้จากหน้าจอเครื่องพิมพ์ หรือ Router Admin

   เปิด Toggle "เสิร์ฟเวอร์ปริ้นจิ๋ว (Direct Print)"
     → เปิดเพื่อพิมพ์ตรงโดยไม่มี dialog เด้ง

   Print Server URL
     ┌─────────────────────────────────────────────────┐
     │ ใช้จากคอมที่รัน .bat  → http://localhost:3001   │
     │ ใช้จากมือถือ/คอมอื่น → http://192.168.1.5:3001 │
     │   (ใช้ IP ที่เห็นใน cmd บรรทัด Network)         │
     └─────────────────────────────────────────────────┘

4. กด "บันทึก"
5. กด "ทดสอบพิมพ์" — ถ้าได้ยินเสียงตัดกระดาษ แสดงว่าเชื่อมต่อสำเร็จ


================================================================
  การหาค่า IP เครื่องพิมพ์
================================================================

วิธีที่ 1 — ดูจากหน้าจอเครื่องพิมพ์
  กดปุ่ม Feed ค้างไว้ 3 วินาที → เครื่องจะพิมพ์ใบ config
  มองหาบรรทัด "IP Address" หรือ "IPv4"

วิธีที่ 2 — ดูจาก Router
  เปิด 192.168.1.1 หรือ 192.168.0.1 ใน browser
  ดูในหน้า Connected Devices / DHCP Clients

วิธีที่ 3 — พิมพ์ Self-Test
  เครื่องพิมพ์ส่วนใหญ่: กดปุ่ม Feed ค้างแล้วเปิดเครื่อง
  IP จะแสดงในใบ Self-Test


================================================================
  แก้ปัญหาเบื้องต้น
================================================================

ปัญหา: กด "ทดสอบพิมพ์" แล้วขึ้น "เชื่อมต่อ Print Server ไม่ได้"
  แก้:  เปิดไฟล์ เริ่มระบบปริ้นเตอร์.bat ค้างไว้ก่อน
        ตรวจสอบว่า Print Server URL ตรงกับ IP ใน cmd

ปัญหา: เปิด .bat แล้วขึ้น "node ไม่ใช่คำสั่งที่รู้จัก"
  แก้:  ติดตั้ง Node.js ก่อน จาก https://nodejs.org
        แล้วรีสตาร์ตคอมพิวเตอร์

ปัญหา: npm install แล้วขึ้น error
  แก้:  ลบโฟลเดอร์ node_modules แล้วรัน npm install ใหม่
        หรือลอง: npm install --legacy-peer-deps

ปัญหา: Print Server เริ่มได้ แต่พิมพ์ไม่ออก
  แก้:  ตรวจสอบ IP เครื่องพิมพ์ว่าถูกต้อง
        ตรวจว่าเครื่องพิมพ์เปิดอยู่และอยู่ WiFi เดียวกัน
        ลอง ping IP เครื่องพิมพ์ใน cmd:  ping 192.168.1.100

ปัญหา: ตัวอักษรภาษาไทยเป็นกล่องสี่เหลี่ยม / อักขระแปลก
  แก้:  ตั้งค่า Code Page ของเครื่องพิมพ์เป็น TIS-620 / CP874
        ดูคู่มือเครื่องพิมพ์ในส่วน "Character Set" หรือ "Code Page"

ปัญหา: มือถือเรียก Print Server ไม่ได้
  แก้:  ตรวจสอบ Firewall ของ Windows
        ไปที่ Control Panel → Windows Defender Firewall
        → Allow an app through firewall → เพิ่ม Node.js
        หรือ: เปิด port 3001 ใน Inbound Rules

  โครงสร้างไฟล์

print-server/

  package.json     — รายการ dependencies
  node_modules/    — โฟลเดอร์ library (สร้างหลัง npm install)

ไฟล์ในโฟลเดอร์หลัก:
  เริ่มระบบปริ้นเตอร์.bat  — ดับเบิลคลิกเพื่อเปิด Print Server


================================================================
  Port ที่ใช้
================================================================

  3001  — HTTP API (รับคำสั่งจากเว็บแอป)
  9100  — TCP RAW ESC/POS (ส่งไปยังเครื่องพิมพ์)
